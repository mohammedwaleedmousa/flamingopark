## 1) إشعارات من الأدمن للعملاء (داخل التطبيق + واتساب)

**قاعدة البيانات** — إنشاء جدول `customer_notifications`:
- `customer_id`, `title`, `body`, `type` (info/order/promo)، `is_read`, `link`, `created_at`
- RLS: العميل يقرأ إشعاراته فقط، الأدمن ينشئ ويحذف
- تفعيل Realtime للجدول

**واجهات جديدة:**
- `AdminCustomerNotificationsPage` — نموذج إرسال إشعار (اختيار عميل واحد/الكل/حسب البلد، عنوان، محتوى، رابط اختياري) + زر إضافي "إرسال عبر واتساب" يفتح `wa.me`
- تحديث `NotificationsPage.tsx` القائمة لتقرأ من `customer_notifications` بدل الوضع الحالي
- تحديث جرس الإشعارات في Navbar (إن كان يعرض بيانات وهمية)

## 2) صفحة `/account` — فحص وإصلاح

سأفحص كل تبويب/زر في الصفحة (طلباتي، مفضلاتي، عناويني، بياناتي الشخصية، تسجيل الخروج، تغيير كلمة المرور...) وأصلح ما لا يعمل:
- ربط "طلباتي" بجدول `orders` عبر `customer_id`
- تفعيل حفظ/تعديل الملف الشخصي عبر `profiles`
- تفعيل إدارة العناوين المحفوظة إن كانت مكسورة
- ربط "المفضلة" بالمخزن الحالي إن كانت لا تعرض
- تفعيل "تغيير كلمة المرور" عبر `supabase.auth.updateUser`

**صفحة أدمن جديدة** `AdminCustomerDetailPage`:
- تعرض كل ما يراه العميل: بياناته، طلباته، مفضلاته، عناوينه، إشعاراته
- زر "إرسال إشعار" و "إرسال واتساب" مباشر
- الوصول من `AdminCustomersPage` بالضغط على العميل

## 3) صفحة تفاصيل المنتج — إعادة تصميم Apple-clean

- تصميم جديد: صور ضخمة يسار (galleries fixed-height sticky)، معلومات مركزة يمين
- طباعة أنيقة، spacing واسع، حذف الشريط السفلي الثقيل واستبداله بـ CTA مثبت أنيق
- إبقاء: الألوان، الأحجام، الإكسسوارات، Reviews، منتجات مشابهة
- تحسين معرض الصور: transitions ناعمة، أرقام صفحات دقيقة
- إضافة **منتج ديمو واحد جديد** (ساعة أو حقيبة فاخرة) بصور Unsplash عالية الجودة لعرض التصميم

## 4) Checkout بـ 4 مراحل

**Wizard جديد بترتيب:**
```
1. المعلومات      → الاسم، الهاتف، البريد
2. العنوان        → البلد، المدينة، الحي، الشارع، ملاحظات
3. الدفع          → طريقة الدفع (COD / تحويل بنكي)، الكوبون
4. المراجعة        → عرض الطلب كامل + زر التأكيد النهائي
```

- Progress Stepper علوي يعرض الخطوة الحالية
- أزرار السابق/التالي مع validation لكل خطوة قبل الانتقال
- حفظ البيانات في state واحد؛ عند التأكيد النهائي فقط يُنشأ الـ order
- الحفاظ على كل المنطق الحالي (WhatsApp PDF، الأرشيف، إشعار الأدمن)
