# خطة التنفيذ الشاملة

نظراً لحجم العمل الكبير سأقسمه إلى **3 مراحل** متتابعة. كل مرحلة يتم تنفيذها وتسليمها قبل الانتقال للتالية، حتى تستطيع المراجعة والتأكد من جودة كل قطعة.

---

## المرحلة 1 — التنقل والروابط المستقلة (الأساس)

**الهدف:** السايد بار هو مركز التنقل، وكل صفحة لها Route مستقل.

- إعادة تصميم **Sidebar** بشكل احترافي (iOS/Glass) داخل `Navbar.tsx`:
  - أقسام مرئية: التسوق / الحسابي / المساعدة
  - تنقل متعدد المستويات (Accordion): الأقسام الرئيسية → الفرعية → الماركات
  - أيقونات Lucide لكل عنصر + Animation سلس
  - حالة المستخدم (مسجّل/زائر) أعلى السايد بار
- إنشاء **Routes مستقلة جديدة** في `App.tsx`:
  - `/categories` صفحة الأقسام
  - `/best-sellers` الأكثر مبيعاً
  - `/new-arrivals` الجديد
  - `/account` صفحة الحساب
  - `/cart` السلة (صفحة كاملة بدل الـDrawer فقط)
  - الإبقاء على: `/products`, `/product/:slug`, `/offers`, `/favorites`, `/checkout`
- إنشاء صفحات جديدة خفيفة تستخدم نفس بنية `ProductsPage` لكن بفلتر مسبق.

## المرحلة 2 — الفلاتر وصفحة المنتجات

- في `ProductsPage.tsx`:
  - **Sidebar فلاتر** على اليسار (Sheet على الموبايل):
    - Category + Subcategory (Accordion)
    - Price Range (Slider مزدوج)
    - Switch: "العروض فقط"
    - Switch: "المتوفر فقط"
  - **Sorting**: الأحدث / الأعلى تقييماً / السعر تصاعدي/تنازلي / الأكثر مبيعاً
  - **مزامنة URL**: كل الفلاتر في `useSearchParams` (category, sub, min, max, sale, stock, sort)
  - زر "إعادة ضبط" واضح
  - شارات (Chips) للفلاتر النشطة
- تحسين `ProductCard` لعرض التقييم وشارة الخصم.

## المرحلة 3 — تفاصيل المنتج + Auth + Checkout

- **ProductDetailPage**:
  - معرض صور Carousel مع Thumbnails وZoom
  - وصف طويل + Tabs (الوصف / المواصفات / الشحن / التقييمات)
  - اختيار المقاس / اللون / الخامة (Chips)
  - متوسط التقييم بالنجوم + عدد المراجعات
  - قسم **منتجات مشابهة** بنفس الفئة
- **AuthPage**:
  - "نسيت كلمة المرور" + صفحة `/reset-password`
  - Loading state واضح + رسائل خطأ عربية مفهومة (ترجمة Supabase errors)
  - انتقال سلس بين Login/Signup (Tabs)
  - تأمين الجلسة عبر `onAuthStateChange` + `getUser()` للتحقق
- **CheckoutPage** Step-by-step:
  - Step 1: تفاصيل العميل والشحن
  - Step 2: طريقة الدفع
  - Step 3: مراجعة وتأكيد
  - ملخص طلب ثابت على الجانب

## ملاحظات تقنية
- لا تغيير في schema قاعدة البيانات (الحقول الحالية كافية: `sizes`, `colors`, `materials` يمكن إضافتها كـ JSONB إن لزم — سأتحقق أولاً).
- استخدام `useSearchParams` للفلاتر بدل state محلي.
- الحفاظ على الـ design tokens الحالية (أبيض/أسود/رمادي فاتح، خط Playfair).
- لن أكسر أي مسار قائم؛ فقط أضيف Routes جديدة وأحسّن الموجود.
